Story Behind the Data

Why this matters: Zara operates in a fast-fashion environment where demand patterns change rapidly. Understanding historical sales trends enables better inventory planning, operational benchmarking, and data-driven decision-making.

Data sources: Publicly available Zara sales dataset containing aggregated monthly sales data across stores and products from 2013 to 2018.

Methodology: Data was cleaned and aggregated using Python (pandas). Descriptive and diagnostic analytics were applied. Interactive visualisations were created using Plotly and embedded into a single-page HTML dashboard.

Interactive Visualisations

Business Insights & Recommendations

  • Sales show a steady upward trend over the analysis period.
  • A small subset of products contributes disproportionately to total sales.
  • High-performing store clusters significantly outperform others.
  • Seasonality plays a clear role in monthly sales fluctuations.

Recommendations

  • Prioritise inventory planning around top-selling products.
  • Use high-performing stores as operational benchmarks.
  • Align promotions with seasonal demand cycles.

Risks & Next Steps

  • Analysis is based on historical, aggregated data.
  • Future work should include pricing and promotion variables.
  • Predictive forecasting models can enhance decision support.

Technical Documentation

Tools: Python (pandas), Plotly, HTML, CSS

Data Governance: Dataset contains no personal data and is GDPR-compliant.

Ethics: Analysis uses aggregated sales data only, ensuring privacy and ethical use.

AI Usage Reflection

Artificial Intelligence tools were used throughout this project as a learning and productivity aid. ChatGPT supported code debugging, visualisation design, and structuring analytical outputs. One key learning outcome was understanding how to translate analytical requirements into precise prompts and critically evaluate AI-generated suggestions.

Challenges included embedding interactive charts into a single self-contained HTML file and aligning outputs with strict technical constraints. These were resolved through iterative testing and manual refinement. AI was not used to fabricate data or automate conclusions. All insights and recommendations were independently validated.

This reflects responsible and ethical AI usage aligned with academic integrity standards.
